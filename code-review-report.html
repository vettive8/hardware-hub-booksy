02 - Current build

One image, one process, one SQLite file.

Vue 3 + Vite->FastAPI /api->SQLite /data/hardware_hub.db+OpenRouter (optional)

The Docker build compiles Vue, copies it into the Python image, and serves both through FastAPI. SQLite is a library and file, not a separate database server or container.

Verified strengths

  • Per-record seed quarantine preserves bad source evidence.
  • Database constraints and conditional updates protect rental state.
  • Server-side role and ownership checks back the UI.
  • Deterministic audit rules remain when OpenRouter fails.
  • Docker, health check, CI, and visible browser tests exist.

Deliberate boundaries

  • Single process and single replica while using local SQLite.
  • Known demo credentials are assessment-only.
  • Bearer token lives in localStorage as a documented shortcut.
  • Paid provider verification remains an explicit command.
  • No live deployment has yet been verified.

04 - Demo identity

Convenient for reviewers, not careless by design.

Reject: guest1-10 / guest

Ten shared, weak accounts add no requirement coverage, conflict with the existing eight-character validation, make actions hard to attribute, and teach the wrong security habit.

Approve with a demo-mode gate

Keep one admin and one member (optionally the assigned John Doe record). Seed them only when DEMO_MODE=true. In production mode, require a strong secret and an explicit admin bootstrap path.

Recommended assessment experience

Show the two demo roles on the login page and in README so the reviewer can test permissions quickly. Keep recognizable, eight-plus-character passwords for the temporary assessment environment. Label them public demo credentials and never reuse them elsewhere.

05 - Deployment decision

Choose the platform that fits the current architecture.

Scores are for this time-boxed assessment, not universal platform rankings.

PlatformFit nowSQLite persistenceChange requiredDecision
Railway9/10Attach one volume at /data; one replica.Set secrets, demo mode, volume, domain; deploy existing Dockerfile.Recommend
Vercel4/10Vercel explicitly does not support local SQLite persistence in Functions.Split frontend/backend and move data to managed Postgres/Turso.Defer
Netlify3/10Functions execute in an ephemeral runtime.Split architecture and add an external API/database.Defer
AWS6/10ECS/Fargate can mount persistent EFS; task storage alone is ephemeral.ECR, ECS, IAM, VPC, load balancer, EFS or RDS, monitoring.Overkill now

Recommended release shape

Railway + existing Dockerfile + volume at /data + one replica. FastAPI serves the built Vue app on the same origin. Set DATABASE_PATH=/data/hardware_hub.db, a generated SECRET_KEY, DEMO_MODE=true for the assessment, and the OpenRouter inference key as a Railway secret.

Know the trade-off

Railway documents that replicas cannot be used with volumes and redeploys have brief downtime. That is honest and acceptable for a recruitment demo. A real multi-instance product should migrate to managed PostgreSQL rather than share a SQLite file.
